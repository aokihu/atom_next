Milestone - 0.7
---------------

# 目标
本次里程碑的目标是补齐 Transport 对 Provider 和 Model 配置错误的处理

- [ ] 在 Transport 中处理 Provider 配置错误
- [ ] 在 Transport 中处理 Model 配置错误
- [ ] 输出更明确的错误信息,方便定位问题

# 当前情况
- 当前配置解析阶段对 Provider 和 Model 的命名错误已经改为温和处理
- 配置文件中的相关字段即使存在疑似错误,也不会阻断系统启动
- 当前阶段只会在配置解析时输出告警信息

# 后续方向
- Provider 和 Model 的最终可用性判断将下沉到 Transport
- Transport 负责在真实创建模型、请求模型时识别这些错误
- 当前文档只记录目标和现状,具体处理方案不在本里程碑中展开
