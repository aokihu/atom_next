Milestone - 0.7
---------------

# 目标
本次里程碑的目标是补齐 Transport 对 Provider 和 Model 配置错误的处理

- [x] 在 Transport 中处理 Provider 配置错误
- [x] 在 Transport 中处理 Model 配置错误
- [x] 输出更明确的错误信息,方便定位问题

# 当前情况
- 当前配置解析阶段只负责校验 `provider/model` 的字符串结构
- 对疑似错误的 Provider 和 Model 命名只输出告警,保留原值到运行阶段
- Provider 和 Model 的最终可用性判断已经下沉到共享 Transport 模型创建链路
- 对话 Transport 和 watchman 编译链路共用同一套 Provider / Model 校验逻辑
- 错误信息会带上具体配置路径,方便直接定位问题

# 后续方向
- 后续新增 provider 时继续复用当前共享模型工厂和错误语义
- 如需支持更多 Provider 原生能力,应在不破坏当前无状态请求链路的前提下扩展
