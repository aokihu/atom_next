## Watchman服务

### 功能说明

**watchman**服务是监视并更新系统提示词的服务,在`{workspace}`目录中会存放一些用户自己的提示词文件`AGENTS.md`,这些提示词会被`Runtime(Core)`合并到一起提交给`Transport(Core)`.
出于安全的考量,用户的提示词不能直接合并,需要通过预编译的方式检查用户的提示词文件中是否包含了违反系统安全准则的内容.
编译的过程是通过将用户提示词文件内容上传到**LLM**进行检查,然后从**LLM**返回的安全的用户提示词存入到内存中,当`Runtime(Core)`需要的时候提供使用.
用户可以随时更改提示词文件,因此**watchman**服务将会监视这些提示词文件的变动,并及时的编译与更新内存中的提示词数据

### 开发指南

- 存放在目录`src/services/watchman`
- 该服务将会派发一个**worker**用来长期监视文件更改
- 编译成功后,在`{workspace}/watchman.json`中记录编译的原信息,方便之后查询用户提示词文件是否更新的时候作为参考
- 编译成功后,在`{workspace}/compiled_prompts`目录中存放编译后的提示词文件,避免在提示词文件未改动的情况下二次编译
- 编译后的**安全的用户提示词文件**命名用**原始用户提示词文件**的`hash`值命名,比如`{workspace}/compiled_prompts/{origin_prompt_file_hash}.md`
- 对用户的提示词文件做*hash*检查,同样的*hash*提示词文件如果在`watchman.json`找到,则不需要再次编译,否则就需要对用户提示词编译

### 对外开发接口

1. `getAgentsPrompt()` 获取到`AGENTS.md`编译后安全的提示词内容
