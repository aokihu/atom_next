# 完善主题系统

- 建立一套可以由用户自己配置的主题系统
- 用户可以在`{workspace}/themes`目录中存放自定义主题配置文件
- 用户可以在`{workspace}/config.json`中通过设置`theme`字段来设置使用主题

## 主题加载流程

1. 检查`{workspace}/config.json`中`theme`是否设置
2. 如果有设置,先检查是不是内置主题,然后检查是不是用户自己配置的主题
3. 如果用户配置主题与内置主题重名,用户配置主题优先
4. 将主题动态加载到TUI

## 潜在风险

- 需要与现有的`bootstrap`和`runtime service`搭配
- 在更新`bootstrap`和`runtime service`时,代码未达到重构标准时,尽可能最小程度的修改已有的稳定代码

# 优化用户输入UI

- 用户输入的Label从`Input`改成`User`
- 去除border的样式设计
- 用户输入的行数为行
- 只有在终端整体高度低于30行时,用户输入才改为1行显示
- 启动后保持用户输入始终处于用户输入聚焦状态

# 优化信息输出UI

- 信息输出只显示回答的消息,中间的通讯过程以及其他的TUI交互状态信息都不显示
- 通讯等待过程中使用"骨架"的方式显示进度
