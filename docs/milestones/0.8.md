# 目标1

Intent Request 功能是LLM返回消息后,在信息结尾附加的内容,这部分内容是LLM向本地请求的指令,比如请求获取记忆内容,请求保存记忆内容,或者请求技艺(SKILL)说明.

Intent Request 是一种非硬性的请求,由`Runtime(Core)`进行处理,

`Runtime(Core)`会对请求指令以及内容进行安全检查,对于不安全的请求`Runtime(Core)`会拒绝处理.

安全的请求在经过`Runtime(Core)`处理后,在下一次提交的时候将这些请求的结果加入到**Context**中,实现消息反馈闭环.

通过这种意图请求的方式,Agent可以实现动态记忆,动态加载技能的能力

# 目标2

## 现状描述
现在的会话只支持一轮,对于会话输入内容不多并且输出内容也不多的时候可以正常运行.
但是当输入的内容太多或者输出内容太多的时候,有可能因为超过了Token的最大限制导致会话失败.

## 解决方案
对于长会话的解决方法采用一个会话多轮提交实现.
也就是每一次提交到LLM的会话需要严格限制输出的Token数量,然后将一次会话拆分为多轮会话实现.这些拆分是在`Core`内部执行,对外暴露的时候只是一次对话

`Runtime(Core)`在接收到相对应的`Intent Request`请求的时候,会自动生成一个新的队列任务,然后插入到**Active Queue**队列中,保证连续会话的优先级是最高的,是最先被执行的

## 依赖
本功能的实现依赖`Intent Request`功能,LLM将会主动请求**继续会话**.我们不会严格限定死LLM应该一次会话输出多少Token,但是会给他一个参考值,当输出的Token要达到参考Token数量的时候,LLM应该通过`Intent Request`请求将会话拆分
