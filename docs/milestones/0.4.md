## 目标
[ ] 完善启动模式契约
[ ] 统一客户端协议目标
[ ] 实现API session测试

## 完善启动模式契约

- 启动模式的参数为`--mode`
- 启动模式分为`tui` `server` `both`三种
- `both`是默认的启动方式,不设置`--mode`参数时候默认`both`
- `tui`表示只启动tui client,需要搭配参数`--server-url`一起使用
- `server` 表示只启动 server

### 当前里程碑实现

- TUI在当前里程碑中只作为一个占位,不实现具体功能,具体功能在里程碑`0.5`中实现
- tui client 将作为一个子进程被启动,不要直接与服务进程强耦合
- 如果`--mode=tui`,则在启动tui client之后主进程立即结束,子进程与主进程生命周期解耦

## 统一客户端协议目标

- 参数`--server-url`目前只接受`http`协议,不支持`websocket`协议
